# Logs Fundamentals

## Overview

Logs are digital records of activity generated by operating systems, applications, users, and network devices.

They act as digital footprints that help security teams monitor activity, investigate incidents, troubleshoot issues, and identify malicious behaviour.

---

# Why Logs Matter

Logs are critical for:

- Security monitoring and threat detection
- Incident investigation and root cause analysis
- System and application troubleshooting
- Performance monitoring
- Auditing and compliance requirements

Logs often provide the evidence needed to reconstruct attacker activity and determine what occurred during a security incident.

---

# Common Log Types

| Log Type | Purpose |
|-----------|-----------|
| System Logs | Operating system events, startup/shutdown, drivers, hardware issues |
| Security Logs | Authentication, authorization, account changes, policy changes |
| Application Logs | Application activity, updates, errors, user interactions |
| Audit Logs | User actions, system changes, compliance tracking |
| Network Logs | Incoming/outgoing traffic, connections, firewall activity |
| Access Logs | Access to websites, databases, APIs, and applications |

---

# Log Analysis Fundamentals

Log analysis involves reviewing logs to identify:

- Suspicious activity
- Security incidents
- User actions
- System changes
- Errors and anomalies

Common approaches include:

- Searching for specific indicators
- Filtering relevant events
- Correlating multiple log sources
- Identifying abnormal patterns

---

# Windows Event Logs

Windows stores logs within **Event Viewer**, allowing analysts to investigate system and security events.

### Key Log Categories

| Log | Purpose |
|------|---------|
| Application | Application events, errors, warnings |
| System | Operating system, hardware, drivers, services |
| Security | Authentication, account activity, policy changes |

### Important Event IDs

| Event ID | Description |
|-----------|-----------|
| 4624 | Successful login |
| 4625 | Failed login |
| 4634 | Logoff |
| 4720 | User account created |
| 4722 | User account enabled |
| 4724 | Password reset attempt |
| 4725 | User account disabled |
| 4726 | User account deleted |

### Key Investigation Fields

- Event ID
- Timestamp
- Log Name
- Description

Filtering by Event ID allows analysts to quickly locate specific activities.

# Web Server Access Logs

Web servers record every request made to a website.

Common fields include:

- Source IP address
- Timestamp
- HTTP method (GET, POST)
- Requested URL
- Response status code
- User-Agent information

These logs help identify user activity, web attacks, and suspicious behaviour.

---

# Useful Linux Log Analysis Commands

### `cat`

Display or combine log files.

```bash
cat access.log
cat access1.log access2.log > combined.log
```

### `grep`

Search logs for specific strings, IP addresses, or indicators.

```bash
grep "192.168.1.1" access.log
```

### `less`

View large log files page-by-page and perform searches.

```bash
less access.log
```

Useful shortcuts:
- `Space` → Next page
- `b` → Previous page
- `/pattern` → Search
- `n` → Next result
- `N` → Previous result

# Key Takeaways

- Logs provide the primary source of evidence during investigations.
- Different log types serve different investigative purposes.
- Windows Event Logs are valuable for investigating authentication and account activity.
- Event IDs allow efficient filtering of specific Windows events.
- Web access logs record requests, timestamps, IPs, URLs, and response codes.
- Common Linux utilities such as `cat`, `grep`, and `less` are effective for manual log analysis.
- Effective log analysis helps detect threats, investigate incidents, and identify attacker activity.
