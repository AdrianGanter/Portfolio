# SOC Workbooks and Lookups

## Identity Inventory

Identity inventory provides contextual information about users, service accounts, and their roles within an organisation.

### Information Obtained
- User roles and responsibilities
- Department information
- Privilege levels
- Contact details
- Employment status

### Common Sources
- Active Directory / Entra ID
- Okta
- Google Workspace
- HR platforms
- Internal databases and spreadsheets

### SOC Usage
Identity lookups help determine whether user activity is expected or suspicious by providing business context during investigations.

---

## Asset Inventory

Asset inventory maintains information about organisational systems and endpoints.

### Information Obtained
- Hostnames
- Server purpose
- Asset ownership
- Operating systems
- Asset location
- Business function

### Common Sources
- Active Directory
- EDR/XDR platforms
- MDM solutions
- Internal asset databases

### SOC Usage
Asset lookups help analysts understand what a system is used for and whether observed activity aligns with its intended purpose.

---

## Network Diagrams

Network diagrams provide visibility into organisational network architecture and system relationships.

### Information Identified
- Network segments and subnets
- Internet-facing services
- Firewall boundaries
- VPN infrastructure
- Internal communication paths

### SOC Usage
Network diagrams help analysts:
- Identify affected network segments
- Understand attack paths
- Trace lateral movement
- Determine the purpose of IP addresses and systems
- Assess the impact of suspicious activity

---

## SOC Workbooks

A workbook (also known as a playbook or runbook) is a structured investigation guide used to ensure alerts are analysed consistently and thoroughly.

### Purpose
- Standardise alert investigations
- Reduce analyst error
- Improve investigation consistency
- Assist junior analysts
- Streamline escalation processes

### Typical Workbook Structure

#### 1. Enrichment
Gather contextual information about:
- Users
- Hosts
- IP addresses
- Threat intelligence indicators

#### 2. Investigation
Analyse:
- Relevant logs
- User activity
- System behaviour
- Alert evidence

#### 3. Escalation & Response
Determine:
- True Positive or False Positive
- Required communications
- Escalation requirements
- Remediation actions

---

## Investigation Workflow Components

### Identity Enrichment
- Review user information
- Validate role and responsibilities
- Confirm expected access patterns

### Asset Enrichment
- Identify system purpose
- Determine business criticality
- Review ownership and location

### Threat Intelligence Enrichment
- Investigate IP reputation
- Check domains and URLs
- Review known indicators of compromise (IOCs)

### Log Analysis
- Examine related events
- Build activity timeline
- Validate alert legitimacy

### Escalation Decision
- Close as False Positive
- Escalate as True Positive
- Request additional information if required

---

## Key Takeaways

- Identity inventory provides user context during investigations.
- Asset inventory provides system and infrastructure context.
- Network diagrams assist with understanding attack paths and affected environments.
- Workbooks standardise alert triage and reduce investigation errors.
- Effective alert investigations combine enrichment, analysis, and escalation processes.
- Structured workflows improve investigation quality and consistency across SOC teams.
