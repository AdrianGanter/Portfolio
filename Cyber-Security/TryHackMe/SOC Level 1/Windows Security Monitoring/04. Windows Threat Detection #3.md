# Windows Threat Detection #3

## Overview

This room covers the final stages of a Windows attack, focusing on how attackers maintain long-term access to compromised systems and prepare for future attacks.

### MITRE ATT&CK Tactics Covered

- **TA0011 – Command and Control (C2)**
- **TA0003 – Persistence**
- **TA0040 – Impact**

---

# Command and Control (C2)

## What is Command & Control?

Command & Control (C2) is the communication channel between an infected machine and the attacker's server.

It allows attackers to:

- Execute commands remotely
- Upload/download files
- Deploy additional malware
- Exfiltrate stolen data
- Maintain long-term control

---

## Interactive Access vs C2

Sometimes attackers already have interactive access through:

- RDP
- SSH
- Web Shell
- Remote Console

However, these sessions disappear when:

- The user logs out
- The service is disabled
- The password changes

Therefore attackers usually establish a persistent C2 channel immediately.

---

## Typical C2 Infection Chain

```
Phishing Email / USB
        │
        ▼
Victim executes attachment
        │
        ▼
Downloader launches
        │
        ▼
Downloads hidden malware
        │
        ▼
Hidden malware connects to C2 server
        │
        ▼
Attacker gains persistent remote control
```

---

## Common Malware Locations

Attackers often hide malware in folders such as:

```
C:\Temp\
C:\ProgramData\
AppData\Local\
AppData\Roaming\
```

---

## Detection

Look for:

- Suspicious downloaded ZIP archives
- Newly created executables
- Outbound DNS connections
- Unknown domains
- Network connections from unexpected processes

Useful logs:

- Sysmon Event ID 1 (Process Creation)
- Sysmon Event ID 3 (Network Connection)
- Security Logs

---

# Persistence

## What is Persistence?

Persistence allows malware or attackers to regain access after:

- Reboots
- User logouts
- Password changes
- Temporary network interruptions

Without persistence, attackers lose access once their original session ends.

---

# User Account Persistence

A common persistence method is creating a hidden local administrator account.

Example:

```cmd
net user "mr.backd00r" "P@ssw0rd!" /add
net localgroup Administrators "mr.backd00r" /add
```

PowerShell equivalent:

```powershell
New-LocalUser
Add-LocalGroupMember
```

---

## Detection

### Event ID 4720

**User Account Created**

Investigate:

- Who created it?
- Was it authorised?
- Source IP
- Login time
- Other activity during the session

---

### Event ID 4732

**User Added to Security Group**

Common abused groups:

- Administrators
- Remote Desktop Users

---

### Event ID 4724

**Password Reset**

Attackers may reset dormant accounts instead of creating new ones.

---

# Windows Services Persistence

Attackers create malicious Windows services that automatically start during boot.

Example:

```cmd
sc create BadService binpath= "C:\malware.exe" start= auto
```

---

## Detection

### Sysmon Event ID 1

Detect execution of:

```
sc.exe create
```

---

### Security Event ID 4697

Service Installed

---

### System Event ID 7045

New Windows Service Installed

---

### Process Tree

Look for suspicious children of:

```
services.exe
```

---

# Scheduled Task Persistence

Scheduled Tasks are one of the most common persistence techniques.

Example:

```cmd
schtasks /create /tn BadTask /tr C:\malware.exe /sc onstart /ru SYSTEM
```

---

## Detection

### Sysmon Event ID 1

Look for:

```
schtasks.exe /create
```

---

### Security Event ID 4698

Scheduled Task Created

---

### Process Tree

Look for malware launched by:

```
svchost.exe
   └── Schedule Service
```

---

# Startup Folder Persistence

Programs inside the Startup folder execute whenever a user logs in.

User Startup Folder:

```
C:\Users\<User>\AppData\Roaming\
Microsoft\Windows\
Start Menu\
Programs\
Startup\
```

All Users:

```
C:\ProgramData\
Microsoft\
Windows\
Start Menu\
Programs\
Startup\
```

---

## Detection

### Sysmon Event ID 11

File Created

Monitor file creation inside Startup folders.

Expected parent process:

```
explorer.exe
```

---

# Registry Run Keys

Programs can automatically start by adding registry entries.

Current User:

```
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

All Users:

```
HKLM\Software\Microsoft\Windows\CurrentVersion\Run
```

Example:

```cmd
reg add HKCU\Software\Microsoft\Windows\CurrentVersion\Run ^
/v BadKey ^
/t REG_SZ ^
/d C:\malware.exe
```

---

## Detection

### Sysmon Event ID 13

Registry Value Set

Monitor modifications to:

```
CurrentVersion\Run
```

# Why Attackers Want Persistence

Persistence enables attackers to:

- Maintain long-term access
- Deploy ransomware later
- Move laterally through Active Directory
- Steal more data over time
- Install additional malware
- Build botnets
- Conduct espionage
- Maintain covert access for months

---

# Active Directory

Once a single Windows host is compromised, attackers often attempt to pivot throughout the Active Directory environment.

Goals include:

- Privilege Escalation
- Lateral Movement
- Domain Admin compromise
- Ransomware deployment

---

# SOC Analyst Detection Priorities

Look for:

- Unexpected new local users
- Administrator group changes
- Password resets
- New Windows services
- New Scheduled Tasks
- Startup folder file creation
- Registry Run key modifications
- Suspicious outbound DNS/network traffic
- Unknown executables in Temp/AppData
- Long-running unusual processes

---

# Key Takeaways

- Command & Control provides remote control of compromised hosts.
- Persistence allows attackers to survive reboots and password changes.
- User account creation is commonly detected with **Event ID 4720**.
- Privileged group additions generate **Event ID 4732**.
- Password resets generate **Event ID 4724**.
- Windows Services (**4697 / 7045**) and Scheduled Tasks (**4698**) are common malware persistence methods.
- Startup Folder persistence is detected using **Sysmon Event ID 11**.
- Registry Run Keys are detected using **Sysmon Event ID 13**.
- Detect attacks as early as possible—preferably during **Initial Access**, before Persistence, Lateral Movement, or Impact.
- The biggest threat to corporate Windows environments is **Ransomware**.
