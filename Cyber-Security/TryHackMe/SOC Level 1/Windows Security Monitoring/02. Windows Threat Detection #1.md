# Windows Threat Detection #1

## Initial Access

**Initial Access** is the stage where a threat actor successfully gains access to a target system.

### Common Initial Access Categories

#### Exposed Services
Internet-facing services that attackers scan for weak credentials or vulnerabilities.

**Examples:**
- RDP (Remote Desktop Protocol)
- HTTP / HTTPS (Web Applications)
- SMTP (Mail Servers)
- MSSQL

**Relevant MITRE ATT&CK Techniques**
- **T1133** - External Remote Services
- **T1190** - Exploit Public-Facing Application

---

#### User-Driven Initial Access
Relies on users executing malicious content.

**Examples:**
- Phishing emails
- Malicious attachments
- Malicious links
- Infected USB devices
- Pirated software

**Relevant MITRE ATT&CK Techniques**
- **T1566** - Phishing
- **T1091** - Removable Media

---

## Initial Access via RDP

### Why RDP is Dangerous

Internet-exposed RDP services are continuously scanned by automated bots.

Typical attack flow:

1. Scan for open RDP (TCP 3389)
2. Brute-force credentials
3. Successful authentication
4. Interactive RDP login
5. Post-compromise activity

---

## Detecting RDP Brute Force

### Failed Logons

**Security Event ID:** `4625`

Look for:

- Numerous failed logons
- External Source IPs
- Logon Type **3** or **10**
- Repeated attempts against the same account

Common brute-force target:

```
Administrator
```

---

### Successful Initial Access

**Security Event ID:** `4624`

Identify:

- Account used
- Source IP
- Logon Type

---

### Logon Types

| Logon Type | Meaning | Typical Usage |
|------------|----------|---------------|
| **3** | Network Logon | SMB, file shares, WinRM, PsExec, network authentication |
| **10** | Remote Interactive | Remote Desktop (RDP) session |

### Visualising the Difference

**Logon Type 3**

```
Attacker
    │
 SMB / WinRM / PsExec
    │
    ▼
Target

✔ Authenticated
✔ Accessing network services
✘ No desktop session
```

---

**Logon Type 10**

```
Attacker
    │
     RDP
    │
    ▼
Windows Desktop

✔ Interactive GUI
✔ Explorer
✔ PowerShell
✔ Full desktop session
```

A common attack sequence is:

```
4625 (Failed attempts)
        ↓
4624 Type 3
(Network authentication succeeds)
        ↓
4624 Type 10
(RDP desktop session established)
```

---

### Investigating Further Activity

After detecting the successful RDP login:

1. Copy the **Logon ID** from the successful `4624` event.
2. Search **Sysmon** logs for the same Logon ID.
3. Review all processes started during that session.

This allows you to reconstruct everything the attacker executed after logging in.

## Phishing Initial Access

Phishing remains one of the most common Windows infection vectors.

### Common Attachment Types

Executable files may use extensions such as:

- `.exe`
- `.com`
- `.scr`
- `.cpl`

Windows hides known extensions by default, enabling deceptive filenames like:

```
invoice.pdf.exe
photo.jpg.exe
```

which appear as:

```
invoice.pdf
photo.jpg
```

---

## LNK (Shortcut) Phishing

Attackers frequently use `.lnk` shortcut files to execute hidden commands.

Example payload:

```
LNK
    ↓
PowerShell
    ↓
Download malware
    ↓
Execute malware
```

A malicious shortcut may execute:

```powershell
powershell.exe -c ...

DownloadFile(...)
Start malware
```

Advantages for attackers:

- Bypasses some AV detections
- Looks like a normal shortcut
- Can launch PowerShell invisibly

---

## Detecting Phishing with Sysmon

### Event ID 1 — Process Creation

Track process execution.

Example:

```
Explorer.exe
    ↓
invoice.pdf.exe
```

Useful fields:

- Image
- ParentImage
- ProcessId

---

### Event ID 11 — File Creation

Track downloaded and extracted files.

Typical chain:

```
Browser launches
        ↓
ZIP downloaded
        ↓
Files extracted
        ↓
Executable launched
```

Example timeline:

```
Sysmon Event 1
msedge.exe launched

↓

Sysmon Event 11
invoice.zip downloaded

↓

Sysmon Event 11
invoice.pdf.exe extracted

↓

Sysmon Event 1
invoice.pdf.exe executed
```

---

## LNK Detection Notes

LNK execution leaves minimal evidence.

Instead of:

```
LNK
    ↓
PowerShell
```

Windows often logs:

```
Explorer.exe
    ↓
PowerShell.exe
```

To identify LNK phishing:

- Look for recently downloaded `.lnk` files.
- Correlate preceding file creation events.
- Investigate PowerShell spawned by `explorer.exe`.

## USB Initial Access

Malware can spread through removable media.

Common techniques include:

- Malicious shortcuts (`.lnk`)
- Fake folders
- Fake USB utilities
- Double-extension executables
- Hidden legitimate files

Examples:

```
Photos.exe
```

```
photo.jpg.exe
```

```
RECOVERY.lnk
```

---

## Detecting USB Malware

Look for processes executed from removable drives.

Example:

```
E:\malware.exe
```

instead of

```
C:\Program Files\
```

Indicators:

- Process Image begins with removable drive letters (`E:\`, `F:\`, etc.)
- Explorer.exe launches executable
- Malware drops payloads to disk
- Malware copies itself to additional USB devices

Typical chain:

```
USB inserted
      ↓
Explorer.exe
      ↓
E:\Malware.exe
      ↓
Drops payload
      ↓
Persistence
      ↓
Copies itself to another USB
```

# SOC Investigation Workflow

### RDP

1. Review `4625` failed logons.
2. Identify repeated failures.
3. Locate successful `4624`.
4. Verify **Logon Type 10**.
5. Record Source IP and account.
6. Use **Logon ID** to investigate Sysmon activity.

---

### Phishing

1. Locate downloaded archive or attachment.
2. Track extracted files.
3. Identify executable launched by `explorer.exe`.
4. Review outbound network connections.
5. Determine malware domain or IP.

---

### USB

1. Identify executable launched from removable drive.
2. Track dropped payloads.
3. Search for persistence mechanisms.
4. Identify propagation to additional removable media.
