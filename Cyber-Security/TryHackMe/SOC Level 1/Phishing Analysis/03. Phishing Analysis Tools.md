# Phishing Analysis Tools
## SOC Level 1 Notes for Portfolio

### Overview

This room focused on the tools and workflows used by SOC analysts to investigate suspicious emails. Rather than manually examining every component, analysts can leverage specialized tools to quickly extract indicators, assess reputations, analyze attachments, and safely observe malware behavior.

---

# Email Analysis Workflow

When investigating a phishing email, collect the following artifacts:

## Header Artifacts

- Sender email address
- Sender IP address
- Subject line
- Recipient addresses (To, CC, BCC)
- Reply-To address
- Date and timestamp
- Mail routing information

## Body Artifacts

- Hyperlinks and URLs
- Shortened URLs
- Attachments
- Attachment file types
- Attachment hashes (SHA256)

These artifacts form the foundation for threat intelligence lookups and IOC collection.

# Email Header Analysis Tools

## Google Messageheader

**Purpose:** Analyze raw email headers and visualize mail routing.

### Capabilities

- Extract sender IP addresses
- Review mail delivery path
- Identify authentication issues
- Display header information in an easy-to-read format

**Link:** https://toolbox.googleapps.com/apps/messageheader/

### Use Case

Paste raw email headers to quickly identify:

- Source IPs
- Relay servers
- SPF/DKIM/DMARC issues
- Suspicious routing paths

---

## Microsoft Message Header Analyzer

**Purpose:** Decode and analyze complex email headers.

### Capabilities

- Header parsing
- Routing analysis
- Authentication review
- Timestamp examination

**Link:** https://mha.azurewebsites.net/

### Use Case

Alternative to Google's Messageheader when investigating suspicious mail delivery chains.

# IP & Domain Reputation Analysis

**Purpose:** Gather intelligence on IP addresses.

### Capabilities

- Geolocation lookup
- ASN information
- Hosting provider identification
- Network ownership information

**Link:** https://ipinfo.io/

### Use Case

- Expected locations
- Cloud providers

### Capabilities

- Website screenshots
- Domain information
- Network requests
- Redirect chains
- Technology detection

**Link:** https://urlscan.io/

### Use Case

- Loaded resources

# Attachment Analysis

## Generating File Hashes

Before investigating a file, calculate its SHA256 hash.

### Linux Command

```bash
sha256sum suspicious_file.pdf
```

### Purpose

- Create a unique file fingerprint
- Search threat intelligence databases
- Identify known malware

# Common Phishing Investigation Workflow

## Step 1

Review email headers.

Identify:

- Sender
- Source IP
- Reply-To address
- Authentication failures

---

## Step 2

Analyze email body.

Look for:

- Urgency
- Brand impersonation
- Suspicious links
- Grammar issues

---

## Step 3

Extract URLs.

Use:

- URL Extractor
- CyberChef
- Manual inspection

---

## Step 4

Perform reputation checks.

Use:

- Talos
- VirusTotal
- IPinfo

---

## Step 5

Analyze attachments.

- Calculate SHA256 hash
- Search VirusTotal
- Search Talos

---

## Step 6

Execute safely in sandbox.

Use:

- ANY.RUN
- Hybrid Analysis
- Joe Sandbox

Observe:

- Process creation
- Network traffic
- Registry modifications
- Downloaded payloads

---

# Key SOC Takeaways

- Never trust display names alone.
- Always inspect email headers.
- Investigate shortened URLs before visiting them.
- Generate hashes for all suspicious attachments.
- Use reputation services before executing files.
- Malware sandboxes provide safe behavioral analysis.
- PhishTool centralizes phishing investigations and accelerates triage workflows.
- Effective phishing analysis relies on collecting and correlating multiple indicators rather than trusting a single artifact.
