# Phishing Emails in Action

## Overview
This room focused on analyzing real-world phishing emails to identify the techniques attackers use to impersonate trusted organizations, manipulate users, and steal credentials. Through multiple phishing scenarios, I learned how to identify red flags within email headers, bodies, hyperlinks, attachments, and sender information.

---

## Key Skills Developed

### Phishing Email Investigation
- Examined real phishing email samples designed to imitate legitimate services.
- Identified common indicators of phishing attacks.
- Analyzed email headers, sender addresses, subjects, embedded links, and attachments.
- Investigated attacker techniques used to increase credibility and urgency.

### Email Spoofing Detection
Learned how attackers:
- Use trusted brand names as display names.
- Register unrelated or suspicious domains.
- Create mismatches between display names and actual sender addresses.
- Impersonate companies such as PayPal, DHL, Netflix, Apple, and shipping providers.

Example:
- Display Name: PayPal Support
- Actual Sender: suspicious external domain

---

## Common Phishing Techniques Observed

### Urgency & Fear Tactics
Attackers frequently:
- Claim unauthorized purchases.
- Warn about account suspensions.
- Create fake delivery notifications.
- Use expiration deadlines to pressure victims into acting quickly.

Purpose:
- Reduce critical thinking.
- Encourage immediate interaction with malicious content.

### Brand Impersonation
Attackers replicated:
- Corporate logos
- HTML email templates
- Colours and branding
- Legitimate wording and formatting

Goal:
- Create trust and legitimacy.

### Credential Harvesting
Observed phishing campaigns designed to:
- Redirect victims through multiple websites.
- Present fake login portals.
- Capture usernames and passwords.

Key indicators:
- Suspicious URLs
- Multiple redirects
- Login pages hosted on unrelated domains
- Generic error messages after credentials are entered

---

## Link Analysis

### URL Shortening Abuse
Attackers used URL shortening services to:
- Hide the true destination.
- Evade quick visual inspection.
- Increase click-through rates.

Learned to:
- Investigate redirects safely.
- Validate final destinations before interacting with links.

### Link Manipulation
Observed cases where:
- Visible text appeared legitimate.
- Embedded hyperlinks pointed elsewhere.
- Tracking numbers and buttons redirected to malicious websites.

Always verify:
- Actual hyperlink destination.
- Domain reputation.
- Root domain ownership.

---

## Tracking Pixels

Learned how attackers embed invisible images that:

- Notify attackers when emails are opened.
- Confirm active email accounts.
- Collect recipient engagement data.

Indicators:
- Remote image loading.
- Suspicious image source URLs.
- Hidden tracking elements in HTML source.

Many email providers block these images by default to protect users.

---

## Attachment-Based Phishing

### Malicious PDF Attachments
Observed phishing emails that:
- Delivered malicious links through PDF files.
- Avoided placing suspicious URLs directly inside the email body.
- Used invoices and billing notices as lures.

### Suspicious Document Types
Investigated:
- PDF attachments
- Microsoft Word Template (.dot) files
- Excel spreadsheets (.xlsx)

Red flags included:
- Unusual attachment types
- Embedded hyperlinks
- Inconsistent content
- Requests for urgent action

---

## Malware Delivery Through Attachments

Analyzed phishing campaigns where:

- Excel documents contained malicious links.
- Clicking links attempted to download executable payloads.
- Attachments served as the initial infection vector.

Potential attacker objectives:
- Establish persistence
- Deploy malware
- Steal credentials
- Exfiltrate sensitive data
- Deliver ransomware

---

## Email Analysis Methodology

When analyzing phishing emails, I learned to:

1. Verify sender identity.
2. Compare display names against actual email addresses.
3. Review subject lines for urgency tactics.
4. Inspect hyperlinks before clicking.
5. Analyze attachments separately.
6. Check for spoofing indicators.
7. Review raw email source code.
8. Identify tracking pixels and redirection chains.
9. Look for credential harvesting attempts.
10. Document indicators of compromise (IOCs).

## Key Takeaway

This room strengthened my ability to investigate real phishing emails from a SOC analyst perspective. I developed practical skills in identifying spoofed senders, analyzing malicious links, investigating suspicious attachments, recognizing credential harvesting attempts, and documenting phishing indicators that can be used to improve organizational detection and response capabilities.
