# Elastic Stack (ELK): The Basics

## Overview

Completed introductory training on the Elastic Stack (ELK) and its use for log analysis, investigation, data visualization, and dashboard creation.

## Core Understanding

Elastic Stack is a platform used to collect, process, store, search, and visualize large volumes of data. While not traditionally a SIEM, it is commonly used by security teams for log analysis and investigations.

### Main Components

- **Elasticsearch** – Stores and indexes data for fast searching and analysis.
- **Logstash** – Collects, parses, and normalizes incoming log data.
- **Beats** – Lightweight agents that collect and ship endpoint or network data.
- **Kibana** – Web interface used to search, investigate, visualize, and create dashboards.

### Data Flow

Beats → Logstash → Elasticsearch → Kibana

## Kibana Fundamentals

Explored Kibana as the primary interface used by analysts to investigate logs.

Key features used:

- Discover tab for log exploration

- Field filtering and searching
- Time-based filtering

## Searching with KQL

Learned the basics of Kibana Query Language (KQL):

- Free-text searches
- Field-based searches
- Wildcards (`*`)
- Logical operators:
  - AND
  - OR
  - NOT

Used KQL to narrow datasets, identify specific users, IP addresses, countries, and investigate activity within defined time ranges.
