# Introduction to SOAR

## Overview

Completed introductory training on Security Orchestration, Automation and Response (SOAR) and how it helps Security Operations Centres improve efficiency, reduce manual workload, and streamline incident response.

## Core Understanding

SOAR platforms integrate multiple security tools into a single interface and automate repetitive investigation and response tasks through predefined workflows known as playbooks.

SOAR is designed to address common SOC challenges such as:

- Alert fatigue
- Large volumes of manual work
- Disconnected security tools
- Slow incident response processes
- Limited analyst resources

## Key Concepts

### Orchestration

Orchestration connects multiple security tools and systems into a unified workflow.

Examples of integrated tools include:

- SIEM
- EDR
- Threat Intelligence platforms
- IAM solutions
- Firewalls
- Ticketing systems

This reduces the need for analysts to manually switch between multiple platforms during investigations.

### Automation

Automation allows predefined investigation steps to be executed automatically.

Examples include:

- Checking threat intelligence sources
- Querying historical login activity
- Creating tickets
- Disabling user accounts
- Gathering investigation data

This helps reduce analyst workload and improves response times.

### Response

SOAR platforms can execute response actions directly from a central interface, including:

- Blocking IP addresses
- Disabling accounts
- Creating incidents and tickets
- Triggering containment actions

## Playbooks

A playbook is a predefined workflow that outlines how a specific type of alert or incident should be handled.

Studied examples of:

- Phishing investigation workflows
- CVE and patch management workflows

Playbooks automate routine tasks while still allowing analysts to make important decisions when required.

## Practical Exposure

Worked through a threat intelligence workflow simulation to understand how SOAR automates investigations by integrating multiple tools and executing predefined response actions.

## Key Takeaways

- Understand the purpose of SOAR within a SOC environment.
- Understand the difference between orchestration, automation, and response.
- Understand how playbooks standardize investigations and incident handling.
- Recognize how SOAR reduces manual effort and improves efficiency.
- Understand that SOAR supports analysts rather than replacing them, with human judgement still required for complex investigations.
