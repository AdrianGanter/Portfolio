# Linux Threat Detection #2

## Attack Flow

```
Initial Access
      ↓
   Discovery
      ↓
Ingress Tool Transfer
      ↓
 Persistence
      ↓
 Cryptominer / Botnet / Proxy
```

---

# Discovery

## Common Discovery Commands

### System
```bash
pwd
ls /
uname -a
hostname
env
```

### Users
```bash
whoami
id
w
last
cat /etc/passwd
cat /etc/sudoers
```

### Processes & Network
```bash
ps aux
ip a
ss -tnlp
netstat -tnlp
```

### Virtualisation / EDR
```bash
systemd-detect-virt
pgrep "<edr>"
```

### Credential Hunting

```bash
find / -name .env
find /home -name id_rsa
history | grep pass
```

### Cryptominer Checks

```bash
cat /proc/cpuinfo
lscpu
free -m
```

**SOC Tip:** `whoami` is a common attacker command and makes a good detection rule.

---

# Detecting Discovery

Useful commands:

```bash
ausearch -i -x <command>
ausearch -i --pid <PID>
ausearch -i --ppid <PID>
```

Always investigate:
- Parent process
- User
- Child processes
- Overall context

# Ingress Tool Transfer

Common attacker tools:

```bash
wget
curl
scp
```

Detection:

- `auditd` process logs
- `/var/log/auth.log` (SSH uploads)
- Downloads to `/tmp`
- Suspicious domains/IPs
- AV/EDR alerts

---

# Dota3 Cryptominer

Typical chain:

```
SSH Brute Force
→ Discovery
→ Password / SSH Key Changes
→ SCP Upload
→ Extract Malware
→ Run Cryptominer
→ Scan Internal Network
```

Common attacker commands:

```bash
tar
nohup
scp
wget
curl
```

Common IOCs:

- Hidden `/tmp/.X26-unix`
- `dota3.tar.gz`
- `authorized_keys` modified
- `mdrfckr` SSH key comment
- Internal SSH scans
- XMRig cryptominer

---

# Key Detection Sources

- `/var/log/auth.log`
- `auditd`
- Network traffic
- File creation in `/tmp`
- Antivirus / EDR
