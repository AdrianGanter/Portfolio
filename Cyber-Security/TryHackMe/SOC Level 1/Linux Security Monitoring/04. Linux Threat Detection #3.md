# Linux Threat Detection 3

## Reverse Shell Detection
- Reverse shells provide attackers with a stable interactive shell after initial compromise.
- Common reverse shell methods:
  - `bash`
  - `socat`
  - `python3`
- Reverse shells are a **critical SOC alert** as they indicate active attacker control.

### Detect with auditd
```bash
ausearch -i -x socat          # Find reverse shell process
ausearch -i --pid <PID>       # Trace parent process
ausearch -i --ppid <PID>      # View child commands
```

### Indicators
- Outbound connection to attacker IP.
- Parent process originates from vulnerable application (e.g. web app).
- Child processes include discovery commands:
  - `id`
  - `whoami`
  - `uname -a`
  - `ls`
  - `pwd`

## Privilege Escalation

Attackers often gain root after initial access.

### Common methods
- Exploit vulnerable software (e.g. **PwnKit**)
- Abuse **SUID** binaries
- Stolen SSH keys

### Detection Indicators
- Burst of discovery commands.
- Exploit downloaded into `/tmp`.
- Compilation (`gcc`) and execution of exploit.
- UID changes from low-privilege user → `root`.

### Useful auditd
```bash
ausearch -i -x pwnkit
ausearch -i --ppid <PID>
```

Watch for:
- `wget`
- `curl`
- `chmod +x`
- exploit execution
- `uid=root`

## Persistence

Attackers establish persistence to survive reboots.

### Cron Jobs
Common locations:
- `/etc/crontab`
- `/etc/cron.d/`
- `/var/spool/cron/`

Look for:
- `crontab -e`
- suspicious scheduled downloads
- `@reboot` entries

---

### Systemd Services

Attackers create fake services to start malware automatically.

Common locations:
- `/etc/systemd/system/`
- `/lib/systemd/system/`

Look for:
- `systemctl enable`
- `systemctl start`
- new `.service` files

### Detect with auditd
```bash
ausearch -i -f /etc/systemd
ausearch -i -x crontab
```

---

## Account Persistence

### New User Creation
Attackers may create privileged users.

Authentication logs:
```bash
/var/log/auth.log
```

Look for:
- `useradd`
- `usermod`
- Added to `sudo`

Example:
```text
new user created
added to sudo group
```

---

### SSH Key Persistence

Instead of passwords, attackers often add their own SSH public key.

Target file:
```text
~/.ssh/authorized_keys
```

Detect changes:
```bash
ausearch -i -f authorized_keys
```

Note:
- `echo >> authorized_keys` often appears simply as `bash` in audit logs.

---

## Useful Detection Workflow

1. Detect reverse shell.
2. Trace parent process.
3. Build process tree.
4. Review discovery commands.
5. Look for privilege escalation.
6. Check persistence:
   - Cron
   - Systemd
   - New users
   - SSH keys
7. Continue hunting for exfiltration or lateral movement.

# Key SOC Takeaways

- Reverse shells usually indicate hands-on-keyboard attackers.
- Build process trees to understand attacker activity.
- Look for UID changes to detect privilege escalation.
- Monitor persistence via:
  - Cron jobs
  - Systemd services
  - New users
  - SSH authorized_keys
- Linux servers are common entry points into Windows environments.
- Linux ransomware, APT campaigns, and espionage are real-world threats.
- Linux threat detection is an essential SOC skill, even in Windows-heavy environments.
