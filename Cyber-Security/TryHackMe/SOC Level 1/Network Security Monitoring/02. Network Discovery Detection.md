# Network Discovery Detection

## Overview
Network discovery is the reconnaissance phase where attackers identify hosts, ports, services, and potential vulnerabilities before attempting exploitation. Defenders perform similar scans for asset inventory and vulnerability management, so distinguishing legitimate from malicious scanning is essential.

---

# Key Concepts

## Attacker Objectives
- Discover live hosts.
- Identify open ports.
- Determine running services.
- Identify service versions.
- Find exploitable vulnerabilities.

## Defender Objectives
- Maintain asset inventory.
- Reduce attack surface.
- Verify only required services are exposed.
- Patch vulnerabilities.
- Differentiate legitimate scanning from malicious reconnaissance.

---

# Reducing False Positives
- Allowlist authorised internal scanners.
- Use Threat Intelligence to identify known malicious scanners.
- Increase alert severity for known malicious IPs.
- Detect generic scanning behaviour for unknown attackers.

---

# External vs Internal Scanning

## External Scanning
- Source IP: External
- Destination: Organisation's public assets
- MITRE ATT&CK: **Reconnaissance**
- Severity: **Low**
- Typical response:
  - Block offending IP.
  - Continue monitoring.

---

## Internal Scanning
- Source IP: Internal
- Destination: Internal assets
- MITRE ATT&CK: **Discovery**
- Indicates attacker already has a foothold.
- Severity: **High**
- Typical response:
  - Investigate affected host.
  - Escalate.
  - Begin Incident Response.
  - Perform root cause analysis.

---

# Horizontal vs Vertical Scanning

## Horizontal Scan
One source scans:
- Same destination port
- Many destination IPs

Purpose:
- Find hosts exposing a specific service.

Example:
- Scanning TCP 445 across an entire subnet.

Detection:
- Same Source IP
- Same Destination Port
- Multiple Destination IPs

Example subnet:
```
203.0.113.0/24
```

---

## Vertical Scan
One source scans:
- One destination IP
- Multiple destination ports

Purpose:
- Profile a specific host.

Detection:
- Same Source IP
- Same Destination IP
- Multiple Destination Ports

Common ports often targeted:
- 80 (HTTP)
- 445 (SMB)
- 3389 (RDP)

---

# Common Scan Types

## Ping Sweep
Protocol:
- ICMP

Purpose:
- Identify live hosts.

Result:
- ICMP Echo Reply = Host online

Notes:
- Often blocked by firewalls.

---

## SYN Scan
Uses:
- TCP three-way handshake

Flow:
1. SYN →
2. SYN-ACK ← (Port open)
3. Scanner never completes handshake.

Advantages:
- Fast
- Stealthier
- Commonly used by Nmap (`-sS`)

---

## UDP Scan
Uses:
- UDP packets

Behaviour:
- ICMP Port Unreachable → Port closed
- No response → Potentially open

Notes:
- Slow
- Less reliable
- Timeout-based detection

---

# Detecting Network Discovery

Indicators:
- Large number of connection attempts.
- Sequential destination IPs.
- Sequential destination ports.
- High connection rate.
- Same source repeatedly probing assets.

---

# Working with Logs

Useful commands:

Preview logs:
```bash
head -n2 logfile.csv
```

Extract CSV columns:
```bash
cut -d',' -f4 logfile.csv
```

Count unique values:
```bash
sort | uniq -c
```

Remove quotes:
```bash
sed 's/"//g'
```

---

# Important Lesson

`cut` works well on simple delimiter-separated files.

If a CSV contains **quoted fields with embedded commas (such as JSON)**, `cut` can split fields incorrectly.

Instead, use tools that understand CSV structure:

- `awk` (with FPAT)
- `csvtool`
- `mlr` (Miller)
- Python (`csv` module)

This avoids incorrect field numbering and makes extracting IPs and ports much more reliable.

# Practical Summary

Performed analysis of exported Zeek connection logs by:
- Identifying internal vs external scanning activity.
- Distinguishing Reconnaissance from Discovery using source/destination IPs.
- Detecting horizontal scans (one port → many hosts).
- Detecting vertical scans (many ports → one host).
- Identifying common service ports (80, 445, 3389).

- Learned the limitations of `cut` on complex CSV files containing embedded JSON and when to use proper CSV-aware tools such as `awk`.
