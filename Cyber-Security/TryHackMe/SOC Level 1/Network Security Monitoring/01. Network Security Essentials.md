# Network Security Essentials

## Overview

Network security monitoring focuses on gaining visibility across an environment to detect, investigate, and respond to malicious activity. Effective investigations rely on correlating host and network logs to reconstruct the attack lifecycle.

---

# Enterprise Network Components

| Component | Security Relevance |
|----------|--------------------|
| **Endpoints** | Common initial compromise via phishing or malware. |
| **File & Database Servers** | High-value targets for ransomware and data theft. |
| **Application Servers** | Internet-facing systems vulnerable to exploitation. |
| **Active Directory** | Central authentication system targeted for privilege escalation. |
| **Routers & Switches** | Critical infrastructure that can enable interception or rerouting if compromised. |
| **Firewalls** | First line of defense and primary source of perimeter activity logs. |

---

# Network Visibility

Effective monitoring combines two primary log sources.

### Host-Centric Logs
Provide visibility into activity occurring **on a system**.

Examples:
- Windows Event Logs
- Linux Syslog
- Application Logs
- EDR / Antivirus

Useful for:
- User logins
- Process execution
- Malware activity
- File and service changes

### Network-Centric Logs
Provide visibility into communication **between systems**.

Examples:
- Firewall Logs
- IDS / IPS
- VPN Logs
- Web Proxy Logs

Useful for:
- Port scanning
- Brute-force attacks
- Lateral movement
- C2 communications
- Data exfiltration

---

# Network Perimeter

The network perimeter separates trusted internal systems from the Internet.

Common security controls include:

- Firewalls
- Routers
- DMZ
- VPN Gateways

Monitoring the perimeter helps identify attacks before they reach internal assets.

---

# Common Attack Indicators

| Pattern | Indicates |
|---------|-----------|
| One IP → Many Ports | Port Scanning |
| Many Failed Logins | Brute Force |
| Internal Host → Multiple Systems | Lateral Movement |
| Regular Outbound Connections | C2 Beaconing |
| Large Outbound Uploads | Data Exfiltration |

---

# Typical Attack Lifecycle

```
Reconnaissance
      ↓
Initial Access
      ↓
Credential Access
      ↓
Lateral Movement
      ↓
Command & Control (C2)
      ↓
Data Exfiltration
```

---

# Investigation Workflow

1. Review firewall logs for blocked reconnaissance.
2. Identify suspicious external IPs.
3. Correlate VPN authentication logs.
4. Confirm successful access following failed logins.
5. Track assigned internal IP activity.
6. Review IDS alerts for exploitation and lateral movement.
7. Identify C2 beaconing.
8. Confirm outbound data exfiltration.

# Useful Commands

```bash
head firewall.log
head ids_alerts.log
head vpn_auth.log

grep "BLOCK" firewall.log
grep "ALLOW" firewall.log
grep FAIL vpn_auth.log
grep C2 ids_alerts.log
grep SMB ids_alerts.log

grep <IP> firewall.log
grep <IP> vpn_auth.log
grep <IP> ids_alerts.log

cat firewall.log | grep BLOCK | cut -d' ' -f5 | cut -d: -f1 | sort | uniq -c

cat vpn_auth.log | grep FAIL | cut -d' ' -f3 | sort | uniq -c

tail ids_alerts.log
```
