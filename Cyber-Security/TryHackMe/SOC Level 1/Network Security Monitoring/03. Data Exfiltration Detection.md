# Data Exfiltration Detection

## Overview
Data exfiltration is the unauthorised transfer of sensitive data from an organisation to an external destination controlled by an attacker. It often occurs after a compromise and can use legitimate protocols to blend into normal network traffic.

---

# Exfiltration Lifecycle

1. Discovery & Collection
   - Locate sensitive files and data.

2. Staging
   - Aggregate, compress or encode data.
   - Common methods:
     - ZIP
     - RAR
     - 7z
     - tar
     - Base64
     - Encryption

3. Exfiltration
   - Transfer data via network, cloud services, removable media or covert channels.

4. Command & Control (C2)
   - Coordinate transfers and confirm successful delivery.

---

# Common Indicators of Exfiltration

Look for:

- Large outbound uploads.
- Unusually high outbound traffic.
- Unknown external IPs or domains.
- Large or frequent HTTP POST requests.
- Long or unusual DNS queries.
- Archive creation (ZIP/RAR/7z).
- Suspicious command-line tools:
  - curl
  - wget
  - Invoke-WebRequest
  - rclone
  - awscli
- Many file read events followed by outbound connections.
- Cloud storage uploads.
- USB/removable media usage.

Useful log sources:
- Sysmon
- Windows Security Logs
- Zeek
- Firewall / Proxy logs
- NetFlow
- DNS logs
- Cloud audit logs

# DNS Exfiltration

## How it works

Attackers encode data inside DNS queries because DNS traffic is almost always allowed through firewalls.

## Indicators

- High DNS query volume.
- Long query names (>30–100 characters).
- Base32/Base64-looking subdomains.
- High-entropy/random strings.
- Numerous TXT queries.
- Frequent NXDOMAIN responses.
- Regular beaconing intervals.

---

## Wireshark Filters

All DNS traffic

```text
dns
```

DNS requests only

```text
dns.flags.response == 0
```

Large DNS packets

```text
dns && frame.len > 70
```

Specific suspicious domain

```text
dns && dns.qry.name contains <domain>
```

---

## Splunk Searches

View DNS logs

```spl
index=data_exfil sourcetype=DNS_logs
```

Queries per source IP

```spl
index=data_exfil sourcetype=DNS_logs
| stats count by src_ip
```

Most queried domains

```spl
index=data_exfil sourcetype=DNS_logs
| stats count by query
| sort -count
```

Long DNS queries

```spl
index=data_exfil sourcetype=DNS_logs
| where len(query) > 30
```

# FTP Exfiltration

## How it works

Attackers upload stolen files to an FTP server using legitimate FTP commands.

## Indicators

- USER/PASS authentication.
- STOR upload commands.
- Large FTP transfers.
- Guest accounts.
- Sensitive filenames (.csv, .pdf, .docx).
- Large PASV data channels.

---

## Wireshark Filters

FTP traffic

```text
ftp || ftp-data
```

Login attempts

```text
ftp.request.command == "USER" || ftp.request.command == "PASS"
```

Uploads

```text
ftp contains "STOR"
```

Sensitive CSV files

```text
ftp contains "csv"
```

Large FTP packets

```text
ftp && frame.len > 90
```

Useful investigation:

- Follow → TCP Stream

---

# HTTP Exfiltration

## How it works

Attackers hide stolen data inside normal HTTP POST requests to external servers or cloud services.

## Indicators

- Large POST requests.
- Unknown external domains.
- High outbound byte counts.
- Chunked or multipart uploads.
- Frequent POST activity.

---

## Splunk Searches

HTTP logs

```spl
index=data_exfil sourcetype=http_logs
```

POST requests

```spl
index=data_exfil sourcetype=http_logs method=POST
```

Average upload size by domain

```spl
index=data_exfil sourcetype=http_logs method=POST
| stats count avg(bytes_sent) max(bytes_sent) min(bytes_sent) by domain
| sort -count
```

Large uploads

```spl
index=data_exfil sourcetype=http_logs method=POST bytes_sent > 600
| table _time src_ip uri domain dst_ip bytes_sent
| sort -bytes_sent
```

---

## Wireshark Filters

All HTTP traffic

```text
http
```

POST requests

```text
http.request.method == "POST"
```

Large POST requests

```text
http.request.method == "POST" && frame.len > 500
```

Very large uploads

```text
http.request.method == "POST" && frame.len > 750
```

Useful investigation:

- Follow → TCP Stream

# ICMP Exfiltration

## How it works

Attackers hide encoded data inside ICMP Echo Requests (ping packets).

## Indicators

- Large ICMP payloads.
- Numerous ICMP Echo Requests.
- Regular beaconing intervals.
- High-entropy payloads.
- External ICMP communication.

---

## Wireshark Filters

All ICMP

```text
icmp
```

Echo Requests

```text
icmp.type == 8
```

Large ICMP packets

```text
icmp.type == 8 && frame.len > 100
```

---

# Investigation Workflow

1. Identify unusual outbound traffic.
2. Determine source host and destination.
3. Check transfer size and frequency.
4. Correlate with process and user activity.
5. Inspect packets in Wireshark.
6. Correlate findings in Splunk.
7. Follow TCP Streams where applicable to inspect transferred data.

---

# Practical Summary

Performed analysis of multiple exfiltration techniques by:

- Detecting DNS tunnelling through long, high-volume DNS queries.
- Identifying FTP uploads using authentication events, STOR commands and transferred filenames.
- Detecting HTTP data exfiltration via unusually large POST requests and outbound uploads.
- Identifying ICMP tunnelling through oversized Echo Request payloads.
- Correlating evidence across Wireshark packet captures and Splunk logs.
- Using packet inspection, traffic size, protocol behaviour and destination analysis to identify suspicious outbound data transfers.
