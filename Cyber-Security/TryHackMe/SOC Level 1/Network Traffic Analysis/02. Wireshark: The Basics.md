# Wireshark: The Basics

## Understanding

Wireshark is a network protocol analyser used to inspect and investigate network traffic captured in PCAP files or from live network interfaces. It allows analysts to observe how devices communicate across a network without modifying any packets, making it an essential tool for troubleshooting, incident response and security investigations.

---

## Interface Overview

I learnt how to navigate the main areas of the Wireshark interface:

- Packet List pane
  - Displays a summary of every captured packet.
  - Includes packet number, timestamp, source, destination, protocol and brief information.

- Packet Details pane
  - Breaks down each packet into protocol layers.
  - Allows inspection of Ethernet, IP, TCP/UDP, HTTP and other protocol headers.

- Packet Bytes pane
  - Displays the raw hexadecimal and ASCII representation of the selected packet.
  - Useful when examining payload data or validating packet contents.

---

## Packet Dissection

Every network packet is built in layers, with each protocol adding its own header information.

By examining packet details I learnt how to identify:

- Source and destination MAC addresses
- Source and destination IP addresses
- Transport protocols (TCP/UDP)
- Source and destination ports
- Time To Live (TTL)
- Sequence and acknowledgement numbers
- Payload size
- Application layer protocols such as HTTP

Understanding these layers makes it possible to determine how data travels through the network and where issues or suspicious behaviour may occur.

## Packet Navigation

Efficient navigation is important when analysing large packet captures.

I practised using features including:

- Jumping directly to packet numbers
- Searching for strings and values
- Reading packet comments
- Using Expert Information to identify warnings and anomalies
- Following protocol streams to reconstruct conversations
- Viewing embedded files transferred within captures

These features greatly reduce the time required to investigate large PCAP files.

---

## Display Filters

Display filters allow analysts to focus only on traffic relevant to an investigation without modifying the original capture.

Examples include:

- Filtering by protocol
- Filtering by IP address
- Filtering by TCP or UDP port
- Filtering by packet contents
- Creating filters directly from selected protocol fields using **Apply as Filter**

Effective filtering makes analysing large captures significantly faster and helps isolate suspicious traffic.

## Key Takeaways

- Wireshark is a passive analysis tool that observes network traffic without altering it.
- Every packet contains information at multiple protocol layers, each providing valuable context during investigations.
- Understanding protocol headers is essential for identifying how systems communicate.
- Display filters are one of Wireshark's most powerful features for isolating relevant traffic.
- Efficient packet navigation and filtering dramatically improve the speed and accuracy of network analysis.
- Strong protocol knowledge is just as important as knowing how to operate the tool, as meaningful analysis depends on understanding what the captured traffic represents.
