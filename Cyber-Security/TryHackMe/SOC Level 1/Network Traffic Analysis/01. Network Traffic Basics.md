# Network Traffic Basics

## Overview
Network traffic analysis is the process of capturing, inspecting, and understanding data moving across a network.  
It goes beyond a single tool (like Wireshark) and instead combines:
- Packet inspection
- Log correlation
- Flow/traffic statistics

The goal is to understand what “normal” looks like and quickly identify abnormal or malicious behaviour.

---

## Why Network Traffic Analysis Matters
Network traffic analysis helps to:
- Detect suspicious or malicious activity
- Investigate and reconstruct security incidents
- Validate alerts from security tools
- Monitor performance issues (latency, spikes, outages)

It is especially useful when logs alone are not enough to see the full picture (e.g., actual payload content).

---

## What Can Be Observed in Network Traffic

### Application Layer
- HTTP requests and responses (e.g., file downloads)
- DNS queries and responses (domains, subdomains, record types)
- Actual payloads (if traffic is decrypted or captured fully)

Example insight:
- A user downloading `suspicious_package.zip`
- A DNS TXT record containing encoded commands (possible C2 communication)

---

### Transport Layer (TCP/UDP)
Key data points:
- Source/destination ports
- TCP flags (SYN, ACK, PSH, etc.)
- Sequence and acknowledgment numbers

Security uses:
- Detect session hijacking (unexpected sequence number jumps)
- Identify unusual session behaviour

---

### Internet Layer (IP)
Key data points:
- Source IP / Destination IP
- TTL (Time To Live)
- Fragmentation details (offset, total length)

Security uses:
- Detect fragmentation-based evasion techniques
- Identify overlapping fragments (possible IDS bypass attempts)

---

### Link Layer (MAC / ARP)
Key data points:
- Source and destination MAC addresses
- ARP request/reply behaviour

Security uses:
- Detect ARP spoofing / poisoning
- Identify multiple MACs claiming the same IP

---

## DNS-Based Threat Example (Key Concept)
Attackers can misuse DNS for:
- Command and Control (C2)
- Data exfiltration
- Tunneling traffic inside DNS queries

Example behaviour:
- Many DNS queries to random-looking subdomains
- TXT records carrying encoded instructions

This is known as:
- DNS tunneling / beaconing

---

## Key Attack Techniques

### DNS Tunneling / Beaconing
- Data hidden inside DNS queries or responses
- Often uses subdomains or TXT records
- Used for C2 communication

### Session Hijacking Detection
- Look for abnormal TCP sequence numbers
- Unexpected packet injection into an existing session

### Fragmentation Attacks
- IP packets split into fragments
- Attackers may overlap fragments to bypass inspection systems

### ARP Spoofing
- Attacker sends fake ARP responses
- Tricks devices into sending traffic through attacker

## Network Traffic Sources

### 1. Endpoint Devices
Devices where traffic starts or ends:
- User PCs
- Servers
- Phones, tablets
- Printers and IoT devices

👉 These generate most network traffic.

### 2. Intermediary Devices
Devices that forward traffic:
- Routers
- Switches
- Firewalls
- Proxies

👉 They generate less traffic but are critical for visibility and control.

---

## Network Traffic Flow Types

### North–South Traffic
- Traffic entering or leaving the internal network (LAN ↔ Internet)
- Example: browsing websites, email access
- Closely monitored for threats

### East–West Traffic
- Internal network communication (LAN ↔ LAN)
- Example: file shares, internal authentication
- Important for detecting lateral movement

---

## How Network Data is Collected

### 1. Logs
- Generated by systems and applications
- Example: firewall logs, web server logs, Windows Event Logs
- Limited visibility (often no payload data)

---

### 2. Full Packet Capture (Deep Visibility)
Two main methods:

**Network Tap**
- Physical device inline in network
- Copies traffic passively
- Very accurate, minimal performance impact

**Port Mirroring (SPAN)**
- Configured on switches
- Duplicates traffic to monitoring port
- Easier to deploy but may affect performance under heavy load

---

### 3. Network Flow Statistics
Instead of full packets, captures metadata such as:
- Source/destination IPs
- Ports
- Volume of traffic

Common protocols:
- NetFlow (Cisco)
- IPFIX (vendor-neutral evolution of NetFlow)

Used for:
- Detecting unusual traffic patterns
- Identifying data exfiltration or lateral movement

---

## Tools Commonly Used
- Wireshark (packet inspection)
- tcpdump (command-line capture)
- IDS/IPS tools (e.g., Snort, Suricata)
- Flow analysis tools (NetFlow/IPFIX collectors)

---

## Key Takeaways
- Logs alone are not enough → full packet capture provides deeper visibility
- Different OSI layers reveal different types of security insights
- Many attacks hide in “normal-looking” traffic (DNS, TCP, ARP)
- Understanding traffic flows (North-South vs East-West) is critical for detection
- Network visibility comes from combining logs + packets + flow data
