# Wireshark: Packet Operations

## Understanding

Wireshark provides powerful analysis capabilities beyond simply viewing captured packets. By using built-in statistics, protocol analysis and advanced filtering techniques, analysts can quickly identify patterns, isolate suspicious traffic and investigate network events more efficiently.

---

## Traffic Statistics

I learnt how to use Wireshark's **Statistics** menu to gain an overview of a packet capture before analysing individual packets.

Useful statistics include:

- Resolved hostnames and IP addresses
- Protocol Hierarchy
- Conversations between endpoints
- Unique Endpoints
- IPv4 and IPv6 statistics
- DNS statistics
- HTTP statistics

Reviewing these statistics first helps build an understanding of network activity and identify areas that require further investigation.

---

## Endpoint Analysis

Wireshark can identify every endpoint involved in a capture, including:

- MAC addresses
- IPv4 and IPv6 addresses
- TCP and UDP endpoints

I also learnt how to enable:

- MAC vendor resolution
- IP and port name resolution
- GeoIP mapping (using a MaxMind database)

These features improve readability and help identify the organisations and geographic locations associated with network traffic.

---

## Protocol Statistics

Wireshark provides protocol-specific statistics that allow analysts to quickly examine activity without manually filtering packets.

Examples include:

- IPv4 and IPv6 traffic summaries
- DNS request and response statistics
- HTTP requests, responses and status codes

These statistics provide valuable insight into how services are being used across a network.

---

## Packet Filtering

Filtering is one of Wireshark's most powerful capabilities.

I learnt the difference between:

- **Capture Filters** – applied before traffic is captured to limit what is recorded.
- **Display Filters** – applied after capture to isolate relevant packets without modifying the original data.

In most investigations, capturing all traffic and narrowing the results using display filters provides the greatest flexibility.

---

## Display Filter Syntax

I practised creating display filters using:

### Comparison Operators

- Equal (`==`)
- Not Equal (`!=`)
- Greater Than (`>`)
- Less Than (`<`)
- Greater Than or Equal (`>=`)
- Less Than or Equal (`<=`)

### Logical Operators

- AND
- OR
- NOT

These operators allow multiple conditions to be combined when searching for specific network activity.

---

## Protocol-Based Filtering

I learnt how to filter traffic at multiple layers of the network stack.

### IP Filters

Used to filter traffic by:

- IP address
- Source and destination hosts
- Subnets
- TTL values

### TCP and UDP Filters

Used to filter by:

- Source and destination ports
- Transport protocol
- Sequence numbers
- TCP flags
- Packet length

### Application Protocol Filters

Filtered application layer traffic including:

- HTTP
- DNS

Examples included filtering:

- HTTP GET and POST requests
- HTTP response codes
- DNS queries and responses
- DNS record types

---

## Advanced Filtering

Wireshark supports advanced filtering functions that make investigations more efficient.

I learnt how to use:

- `contains` to search for text within protocol fields
- `matches` to perform regular expression searches
- `in` to search multiple values simultaneously
- `upper()` and `lower()` for case-insensitive searches
- `string()` to convert non-string values for filtering

These functions make it possible to build highly specific search queries when investigating network traffic.

---

## Investigation Workflow

To improve efficiency during investigations, I also learnt how to:

- Save frequently used display filters as bookmarks
- Create one-click filter buttons
- Configure multiple Wireshark profiles with different colouring rules, preferences and filters for different investigation scenarios

These features help streamline repetitive analysis tasks and create consistent workflows.

---

## Practical Skills Gained

Through practical exercises I developed the ability to:

- Use statistical summaries to quickly understand packet captures
- Identify important hosts and network conversations
- Analyse protocol-specific activity
- Differentiate between capture and display filters
- Build complex display filters using logical operators
- Filter traffic across IP, TCP, UDP, HTTP and DNS protocols
- Use advanced filtering functions to locate specific packet data
- Configure reusable filters and investigation profiles for faster analysis

---

## Key Takeaways

- Reviewing packet statistics provides valuable context before analysing individual packets.
- Display filters are essential for efficiently isolating relevant traffic during investigations.
- Protocol-specific statistics can quickly reveal abnormal behaviour or suspicious activity.
- Advanced filtering functions enable highly targeted packet searches.
- Saving filters, profiles and colouring rules improves consistency and speeds up future investigations.
- Combining statistical analysis with effective filtering allows analysts to investigate large packet captures quickly and accurately.
