# Detecting Web Shells

## What is a Web Shell?

A **web shell** is a malicious script uploaded to a web server that allows an attacker to execute operating system commands remotely through a web browser.

Common purposes:
- Initial access via file upload vulnerabilities
- Persistence after compromise
- Remote command execution
- Privilege escalation
- Reconnaissance
- Lateral movement
- Data exfiltration

**MITRE ATT&CK**
- **T1505.003 – Server Software Component: Web Shell**

---

# How Web Shells are Deployed

Attackers require:

- File upload vulnerabilities
- Weak upload validation
- Misconfigured permissions
- Existing access to the server

Common upload mistakes:

- No file extension validation
- No MIME type validation
- No content validation
- Uploading directly into web-accessible directories

Example:

```
/uploads/shell.php
```

Instead of:

```
/uploads/photo.jpg
```

# Anatomy of a Web Shell

Most web shells abuse legitimate programming functions.

### Common PHP Functions

```
shell_exec()
```

```
exec()
```

```
system()
```

```
passthru()
```

Typical workflow:

1. Read URL parameter
2. Store command
3. Execute command
4. Return output

Example:

```
?cmd=whoami
```

# Web Server Log Analysis

Access logs are one of the best sources for detecting web shell activity.

Useful fields include:

| Field | Indicators |
|--------|------------|
| Client IP | External or known malicious IPs |
| Timestamp | Activity outside business hours |
| HTTP Method | Suspicious PUT/POST requests |
| Requested URI | Access to uploaded PHP files |
| Response Code | 200 after upload attempts |
| User-Agent | curl, wget, outdated browsers |
| Referrer | Missing or empty |

---

# Suspicious HTTP Methods

| Method | Legitimate Use | Possible Abuse |
|---------|----------------|----------------|
| GET | Retrieve resource | Recon or web shell interaction |
| POST | Submit data | Upload or control web shell |
| PUT | Upload/replace files | Upload web shell |
| DELETE | Remove resources | Remove evidence |
| OPTIONS | Discover supported methods | Reconnaissance |
| HEAD | Retrieve headers | File discovery |

## Query Strings

The **query string** associates values with URL parameters.

Example:

```
shell.php?cmd=whoami
```

Red flags:

- `cmd=`
- `exec=`
- Long parameters
- Encoded payloads
- Base64 strings

---

## Missing Referrer

May indicate:

- Direct access to a web shell

Although missing referrers are not always malicious.


# Key Takeaways

- Web shells provide attackers with remote command execution through a web server.
- They commonly exploit insecure file upload functionality.
- Web shells are both an **initial access** and **persistence** technique (MITRE **T1505.003**).
- Indicators include suspicious HTTP methods, repeated requests, external IPs, unusual User-Agents, missing referrers, and query strings containing commands.
- Correlate web logs, auditd logs, file system artifacts, and network captures for the most reliable detection.
- Search web roots for newly created or modified executable files and inspect them for dangerous PHP functions such as `eval()`, `shell_exec()`, `exec()`, `system()`, and `passthru()`.
- Network captures can expose uploaded web shell source code, executed commands, and attacker activity that access logs may not record.
