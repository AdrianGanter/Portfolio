# Web Security Essentials

## Why Web Security Matters

- Modern applications have largely shifted from desktop software to web-based applications due to:
  - Greater accessibility
  - Platform independence
  - Faster deployment and updates
  - Cloud computing and SaaS adoption

- Because web applications are publicly accessible and often connected to databases and internal infrastructure, they are one of the most common attack surfaces.

### Common Risks

**For Application Owners**
- Internet-facing services require continuous protection.
- Responsible for protecting user data.
- Must defend against constantly evolving threats.

**For Users**
- Compromised web applications can expose personal and financial data.
- Browser compromise may lead to account theft.
- Privacy and identity can be permanently affected.

---

# Web Infrastructure

Every web service consists of three core components:

### Application
Contains the website logic, code, images, stylesheets and functionality.

### Web Server
Receives HTTP requests, processes them and returns responses.

Common web servers:
- Apache
- Nginx
- Microsoft IIS

### Host Machine
The operating system running both the web server and application (typically Linux or Windows).

---

# Request / Response Cycle

A browser sends an HTTP request to a web server.

The server:
1. Receives the request.
2. Processes it.
3. Performs authentication/authorization if required.
4. Returns the requested resource or data.

Attackers commonly abuse this process through:
- Request flooding (DoS/DDoS)
- Access control bypasses
- Injection attacks
- Malicious payload execution

# Securing Web Applications

## Application Security

- Secure coding practices
- Input validation and sanitisation
- Proper error handling
- Role-based access control (RBAC)

---

## Web Server Security

- Enable detailed access logging
- Deploy a Web Application Firewall (WAF)
- Use a Content Delivery Network (CDN)

---

## Host Security

- Apply the Principle of Least Privilege
- Disable unnecessary services
- Close unused ports
- Deploy endpoint antivirus
- Harden the operating system

---

## Security Practices Across All Layers

- Strong authentication
- Regular patch management
- Continuous monitoring and logging

---

# Access Logs

Web servers record every request they receive.

Useful log fields include:
- Client IP address
- Timestamp
- Requested URL
- HTTP method (GET/POST)
- HTTP status code
- User-Agent
- Referrer

These logs allow analysts to:
- Reconstruct user activity
- Detect malicious behaviour
- Investigate security incidents
- Identify attack patterns

---

# Content Delivery Networks (CDNs)

A CDN distributes cached website content across geographically distributed edge servers.

Benefits:
- Faster content delivery
- Reduced latency
- Lower origin server load

Security advantages:
- Masks the origin server IP
- Provides DDoS mitigation
- Enforces HTTPS/TLS
- Frequently integrates a Web Application Firewall

---

# Web Application Firewalls (WAFs)

A WAF inspects HTTP traffic before it reaches the application.

Primary deployment models:
- Cloud-based (Reverse Proxy)
- Host-based
- Network-based

Common detection methods:

### Signature-Based Detection
Matches requests against known attack signatures.

Example:
- Known SQLMap User-Agent
- Known exploit payloads

### Heuristic Detection
Detects suspicious request characteristics.

Example:
- Abnormally long query strings
- Suspicious encoded characters

### Behavioural / Anomaly Detection
Identifies deviations from normal traffic.

Example:
- Rapid repeated login attempts
- Unusual request frequency

### IP Reputation & Geolocation Filtering
Blocks requests from:
- Known malicious IPs
- Unexpected geographic locations

# Antivirus

Antivirus protects the host machine rather than the web application itself.

Common uses include detecting:
- Malware
- Web shells
- Malicious uploads
- Post-exploitation tools

Most antivirus solutions primarily rely on signature-based detection and should be used alongside other security controls as part of a defence-in-depth strategy.

---

# Key Security Concepts

- Defence-in-depth uses multiple overlapping security controls.
- Mitigation refers to reducing or limiting the impact of attacks.
- Patch management keeps software and dependencies updated to reduce exploitable vulnerabilities.
- Logging provides visibility for detection, investigation and incident response.
- Security should be applied across the application, web server and host machine—not just one layer.
