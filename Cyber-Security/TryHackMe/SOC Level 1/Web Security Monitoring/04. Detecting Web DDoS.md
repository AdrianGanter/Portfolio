# Detecting Web DDoS

## Overview

A **Denial-of-Service (DoS)** attack attempts to make a website or application unavailable by exhausting its resources.

A **Distributed Denial-of-Service (DDoS)** attack achieves the same goal but uses many compromised devices (**botnet**) to generate far greater traffic volumes.

**Primary security objective targeted:**

- **Availability** (CIA Triad)

---

# DoS vs DDoS

| DoS | DDoS |
|------|-------|
| Single attacking machine | Multiple compromised devices (botnet) |
| Limited by one system's CPU, RAM and bandwidth | Scales across thousands or millions of devices |
| Easier to block | Much harder to mitigate |

### Botnet

A **botnet** is a collection of compromised computers, IoT devices, and servers controlled by an attacker to launch coordinated attacks.

---

# Layer 7 (Application Layer) DDoS

This room focuses on **Application Layer (OSI Layer 7)** attacks.

Rather than flooding raw network bandwidth, attackers overwhelm the web application itself by targeting expensive operations.

Examples:

- Login pages
- Search functions
- API endpoints
- Shopping carts
- Registration forms

---

# Common Application-Layer DDoS Attacks

| Attack | Description |
|----------|-------------|
| Slowloris | Holds many partial HTTP connections open to exhaust server resources. |
| HTTP Flood | Sends massive numbers of HTTP requests. |
| Cache Bypass | Bypasses CDN caching to force origin server processing. |
| Oversized Queries | Forces expensive searches or large database queries. |
| Login/Form Abuse | Floods authentication and password reset forms. |
| Faulty Input Validation Abuse | Exploits inefficient input handling to consume resources. |

---

# Why Attackers Launch DDoS Attacks

## Financial Loss

Prevent customers from using services.

Example:

- E-commerce during holiday sales.

---

## Extortion

Demand payment to stop the attack.

---

## Hacktivism

Political or social protest.

---

## Distraction

Divert defenders while another attack occurs.

Example:

- DDoS while ransomware or data theft is underway.

---

## Competition

Disrupt a competitor's services.

---

## Denial of Wallet

Force victims to incur cloud or API usage costs.

---

## Reputational Damage

Reduce customer confidence in a business.

Example:

- Game servers crashing at launch.

# Common DDoS Indicators in Web Logs

## High Request Rate

Example:

```
1000 GET /login
```

Large numbers of requests to the same endpoint.

---

## Burst Activity

Many requests occurring within the same second.

Automation often creates unnatural timing patterns.

---

## Suspicious User-Agents

Examples:

```
curl
```

```
Python-urllib
```

Outdated browsers.

Automated tools often identify themselves through User-Agent strings.

---

## Geographic Anomalies

Traffic suddenly arrives from many countries simultaneously.

Often indicates a distributed botnet.

---

## Server Errors

Large increase in:

```
500 Internal Server Error
```

```
502 Bad Gateway
```

```
503 Service Unavailable
```

```
504 Gateway Timeout
```

Frequent **5xx** responses suggest the server is overwhelmed.

---

## Logic Abuse

Attackers intentionally submit expensive requests.

Example:

```
GET /products?limit=999999
```

Large database operations consume significant CPU and memory.

# Investigating DDoS with Logs

Look for:

- Repeated requests
- Same URI
- High request frequency
- Same User-Agent
- Multiple attacking IPs
- Burst timestamps
- Increased 5xx responses

Useful fields include:

- Client IP
- URI
- Timestamp
- HTTP Method
- Response Code
- User-Agent

---

# Using SIEM (Splunk)

SIEM platforms simplify DDoS investigations by:

- Centralizing logs
- Searching fields quickly
- Correlating events
- Visualizing traffic

Useful searchable fields:

- clientip
- uri
- useragent
- status

---

# Splunk Investigation Ideas

Identify:

Most requested URI

Top attacking IP

Most common User-Agent

Number of unique attacker IPs

Peak requests per second

First legitimate client receiving 503 responses

Useful visualizations:

- Timecharts
- Pie charts
- Request counts
- Geographic maps

# Mitigation Strategies

## Secure Development

Applications should:

- Validate input
- Limit query size
- Reject malformed requests
- Prevent expensive operations

---

## CAPTCHA

Blocks automated traffic by requiring users to solve a challenge.

Useful against:

- Bots
- Login abuse
- Form abuse

---

## JavaScript Challenges

Background browser checks verify legitimate users without requiring visible CAPTCHA interaction.

---

# Content Delivery Network (CDN)

A CDN protects web servers by:

- Caching content
- Serving requests from edge locations
- Reducing origin server load

Benefits:

- Faster performance
- Lower latency
- Better DDoS resilience

---

## CDN Load Balancing

Distributes traffic across multiple servers.

Prevents one server from becoming overloaded.

---

# Web Application Firewall (WAF)

WAFs inspect incoming HTTP requests.

Possible actions:

- Allow
- Block
- Challenge

Rules may include:

- Rate limiting
- User-Agent filtering
- IP reputation
- Threat intelligence
- Geographic filtering

---

## Rate Limiting Example

```
IF requests to /login.php > 5 per minute

THEN block originating IP
```

Prevents:

- Login flooding
- Credential stuffing
- Basic DoS attacks

---

# Large-Scale Mitigation

Major providers mitigate enormous attacks using:

- Global Anycast networks
- Edge servers
- Massive bandwidth
- Traffic filtering
- Load balancing

Examples:

- Google mitigated ~398 million requests/second (2023)
- Cloudflare mitigated an 11.5 Tbps attack

---

# Attacker Evasion Techniques

Attackers attempt to bypass protections by:

- Random query parameters

Example:

```
/products?a=abcd
```

This bypasses CDN cache and forces the origin server to generate a fresh response.

Other evasion techniques:

- Rotate User-Agent strings
- Spoof referrers
- Use globally distributed IP addresses
- Rotate source IPs

# Key Takeaways

- **DoS** attacks use a single machine, while **DDoS** attacks use a distributed **botnet**.
- The primary objective is to compromise **Availability**.
- Layer 7 attacks target expensive web application functions rather than raw bandwidth.
- High-value targets include **login**, **search**, **API**, **checkout**, and **registration** endpoints.
- Indicators include high request rates, burst activity, repeated URIs, unusual User-Agents, geographic anomalies, and spikes in **5xx** errors.
- SIEM platforms (e.g., Splunk) make investigations easier by correlating logs and visualizing traffic.
- Effective mitigations include secure coding, input validation, **CAPTCHA**, JavaScript challenges, **CDNs**, **WAFs**, **rate limiting**, and **load balancing**.
- Attackers may attempt to bypass defenses using random query parameters, changing User-Agents, or rotating source IP addresses.
