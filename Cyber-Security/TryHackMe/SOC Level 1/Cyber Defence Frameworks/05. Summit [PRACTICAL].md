# 🔍 TryHackMe – Summit

## 🧠 Overview
This room simulates a real-world **purple team engagement** where I acted as a SOC Analyst responsible for detecting and preventing malware execution.  

The challenge followed the **Pyramid of Pain**, progressively moving from simple indicators (hashes) to advanced behavioural detections (TTPs).  

My approach was iterative:
- Analyse malware in a sandbox  
- Extract indicators or behaviours  
- Build detection rules  
- Validate prevention and repeat  

---

## ⚙️ Workflow I Followed
For each task, I consistently:
1. Retrieved the malware sample from the inbox  
2. Submitted it to the Malware Sandbox  
3. Analysed the output (hashes, network activity, behaviour)  
4. Identified the most appropriate detection method  
5. Created rules in the relevant tool (Hash, Firewall, DNS, Sigma)  
6. Verified the malware was successfully blocked  

---

## 🧪 Detection Progression (Pyramid of Pain in Action)

### 1. Hash Detection (Sample 1)
- Analysed the file in the sandbox  
- Extracted its hash (MD5/SHA256)  
- Added it to the blocklist via **Manage Hashes**  

✅ *Understanding:*  
This is the simplest form of detection, but also the easiest for attackers to bypass by modifying the file.

---

### 2. IP Blocking (Sample 2)
- Reviewed network activity from sandbox results  
- Identified a suspicious outbound IP (C2 server)  
- Created a **Firewall rule** to block traffic to that IP  

✅ *Understanding:*  
Blocking IPs disrupts attacker infrastructure, but can still be bypassed if they rotate servers.

---

### 3. DNS Blocking (Sample 3)
- Analysed DNS queries generated by the malware  
- Identified a malicious domain  
- Blocked it using the **DNS Rule Manager**  

✅ *Understanding:*  
Domains are more resilient indicators than IPs, making them more effective to block.

---

### 4. Host Artefacts (Sample 4)
- Investigated file system activity and system changes  
- Identified suspicious file paths or artefacts  
- Created detection rules based on host-level indicators


✅ *Understanding:*  
This shifts detection onto the endpoint, making it harder for attackers to evade without changing behaviour.

---

### 5. Behaviour-Based Detection (Sample 5)
- Focused on process activity and execution patterns  
- Identified attacker behaviour rather than static indicators  
- Built a **Sigma rule** to detect those behaviours

**Scenario**: reviewed logs over 12 hour period. Noticed malicious IP executing a .bat file to keep a process alive every 1800 seconds @ a consistent 97 bytes. 


**Resolution**: Created a detection rule matching any IP and any port making a connection at 97 bytes with a frequency of 1800 seconds that matched ATT&CK ID Command and Control (TA0011)

✅ *Understanding:*  
Behaviour-based detection is significantly harder to evade and aligns with real SOC practices.

---

### 6. Advanced Detection / TTPs (Final Stage)
- Analysed more complex attack behaviour  
- Combined multiple detection strategies  
- Tuned rules to detect attacker techniques (TTPs)

**Scenario**: A log of activity showed a suspicious file "exfiltr8.log" being created on process cmd.exe.


**Resolution**: Created a detection rule for cmd.exe process that contains in command line: exfiltr8.log and selected ATT&CK ID Exfiltration (TA0010)

✅ *Understanding:*  
At this stage, detection targets **how the attacker operates**, not just what they use.

---

## 🧩 Key Takeaways
- Detection engineering is **iterative and layered**, not one-dimensional  
- Static indicators (hashes, IPs) are useful but limited  
- Strong detection comes from understanding **attacker behaviour**  
- The Pyramid of Pain is a practical framework for prioritising detections  
- Tools like Sigma enable scalable, reusable detection logic

## ✅ Final Reflection
This room reinforced the importance of moving beyond basic IoCs and focusing on **resilient, behaviour-driven detections**.  

Rather than simply stopping one attack, the goal is to make it **costly and difficult for an attacker to retool and try again** — which is exactly what strong SOC operations aim to achieve.
