## File & Hash Threat Intelligence

### Overview

This room focused on the enrichment phase of the SOC triage process, where suspicious files are investigated using file attributes, cryptographic hashes, threat intelligence platforms, and sandbox analysis. The objective was to determine whether a file was benign, suspicious, or malicious by gathering contextual intelligence and mapping observed behaviour to MITRE ATT&CK techniques.

---

### Key Concepts Learned

#### File Path & Filename Analysis

Before analysing a file's contents, valuable indicators can be gathered from its location and naming conventions.

Common attacker tradecraft includes:

- Storing payloads in writable directories such as:
  - `C:\Users\Public`
  - `C:\ProgramData`
  - `C:\Windows\Temp`
- Using double extensions (e.g. `invoice.pdf.exe`)
- Masquerading as legitimate system files
- Using high-entropy or randomly generated filenames
- Employing slight character substitutions to evade visual detection

While these indicators alone do not confirm maliciousness, they provide valuable investigative leads during initial triage.

---

#### File Hashing

File hashes provide a unique fingerprint of a file and remain consistent regardless of filename changes.

Common hash algorithms include:

- MD5
- SHA1
- SHA256

As a SOC analyst, hashes are essential for:

- Identifying known malware
- Tracking files across environments
- Sharing indicators of compromise (IOCs)
- Correlating intelligence across multiple threat platforms

I also reinforced my ability to generate hashes using both Windows and Linux utilities.

---

### Threat Intelligence Enrichment

#### VirusTotal

VirusTotal aggregates detection results from numerous security vendors and provides valuable intelligence about suspicious files.

Important data points include:

- Detection ratios
- Threat classifications
- File properties and metadata
- Digital signatures
- Associated domains and IP addresses
- Behavioural indicators
- Embedded or dropped files

A file's detection count alone should not determine a verdict. Additional context such as vendor consensus, behavioural indicators, and historical submissions must also be considered.

---

#### MalwareBazaar

MalwareBazaar complements VirusTotal by focusing on malware sharing and research.

Useful intelligence includes:

- Malware family classifications
- YARA detection rules
- Threat actor campaign associations
- Sample downloads for deeper analysis

This allows analysts to connect suspicious files to known malware families and adversary activity.

---

### Sandbox Analysis

Static indicators only reveal part of the picture. To understand how malware behaves, files can be executed safely inside a sandbox environment.

Sandbox analysis enables analysts to:

- Confirm malicious execution
- Observe runtime behaviour
- Extract new IOCs
- Identify persistence mechanisms
- Detect network communications
- Map activity to MITRE ATT&CK techniques

Common sandboxing platforms include:

- Hybrid Analysis
- Joe Sandbox

Behavioural analysis often reveals actions that cannot be identified through static analysis alone.

---

### Sandbox Limitations

I also learned that sandbox results should never be treated as absolute truth.

Common limitations include:

#### Evasion Techniques

Malware may detect:

- Virtual machines
- Sandboxes
- Debuggers
- Analysis environments

and alter its behaviour to avoid detection.

#### Time-Based Evasion

Some malware delays execution beyond the sandbox observation period.

#### Encrypted Communications

HTTPS traffic and encrypted payloads can reduce visibility into malicious activity.

#### Fileless Malware

Modern threats may execute entirely in memory using legitimate system tools, leaving minimal file-based evidence.

Because of these limitations, sandbox results should always be validated against additional intelligence sources.

---

### MITRE ATT&CK Integration

A major focus of the room was linking observed malware behaviour to MITRE ATT&CK techniques.

Examples include:

- Persistence
- Privilege Escalation
- Command and Scripting Interpreter (PowerShell)
- Scheduled Tasks
- Registry Modifications
- Process Injection

Mapping activity to ATT&CK provides a standardised method of documenting adversary behaviour and improving detection coverage.

---

### SOC Analyst Takeaways

Through this room I developed a practical workflow for investigating suspicious files:

1. Examine file paths and filenames for suspicious indicators.
2. Generate and validate file hashes.
3. Enrich hashes using threat intelligence platforms.
4. Review vendor classifications and malware family information.
5. Analyse sandbox behaviour and runtime indicators.
6. Extract IOCs and behavioural evidence.
7. Map observed activity to MITRE ATT&CK techniques.
8. Determine whether the file is benign, suspicious, or malicious.

This process demonstrates how SOC analysts transform a simple file alert into actionable threat intelligence that supports detection, investigation, and incident response activities.
