## Phishing Kit Investigation

### Objective
Investigated a compromised website hosting a phishing kit to identify attacker infrastructure, analyze the phishing kit, and recover exposed information.

### Web Enumeration
Began by exploring the exposed `/data` directory of the website to identify publicly accessible files and directories.

![Exposed data directory](https://github.com/user-attachments/assets/4af1553e-998f-4f4e-99e7-a9b630987493)

Discovered a downloadable phishing kit within the directory.

![Phishing kit discovered](https://github.com/user-attachments/assets/f762af66-8c5c-4dae-a395-46c49f3d4a2f)

---

### Phishing Kit Analysis
Downloaded the phishing kit inside a virtual environment and generated its SHA-256 hash for analysis.

![SHA-256 hash](https://github.com/user-attachments/assets/32c957a0-3868-4c39-a1fa-ee75668ad5fc)

Submitted the hash to VirusTotal.

Results:
- Associated with phishing activity.
- Additional classifications included **Trojan** and **HackTool**.
- Archive contained **49 files**.

![VirusTotal analysis](https://github.com/user-attachments/assets/f4c4e329-63db-484a-92bf-d446fe730fbf)

---

### Credential Harvesting Investigation
Located the file:

```
/data/Update365/log.txt
```

The log contained credentials submitted through the phishing page.

Analysis showed one user had submitted credentials multiple times:

- `michael.ascot@swiftspend.finance`

![Credential log](https://github.com/user-attachments/assets/14af8235-c713-456c-8499-29b76d69f745)

![Credential submissions](https://github.com/user-attachments/assets/da186561-7845-47f4-bf3c-39bdd42c1e3f)

---

### Source Code Review
Extracted the phishing kit and examined the PHP source files.

Located `submit.php`, which revealed the attacker-controlled email address receiving harvested credentials:

```
m3npat@yandex.com
```

![submit.php analysis](https://github.com/user-attachments/assets/1580d71f-1a88-410f-9718-8c43ac08d0f4)

---

### Flag Discovery
The final task required locating a hidden `flag.txt` file within the phishing website.

Initially investigated multiple directories within the extracted phishing kit before realising the simplest solution was to directly browse to:

```
/office365/flag.txt
```

The file contained a Base64-encoded string which was decoded and then reversed using CyberChef to reveal the final flag.

![Flag discovery](https://github.com/user-attachments/assets/cc308894-592d-48af-b356-2155bbca2647)

---

### Key Takeaways
- Performed web directory enumeration to identify exposed attacker resources.
- Investigated and analysed a phishing kit in an isolated virtual environment.
- Verified malware using SHA-256 hashing and VirusTotal.
- Examined harvested credential logs to identify compromised users.
- Reviewed PHP source code to uncover attacker infrastructure.
- Used CyberChef to decode and manipulate encoded data.
- Reinforced the importance of checking straightforward file paths before assuming a more complex solution—sometimes the simplest URL is the correct one.
